# 对称加密与非对称加密

## 1. 对称加密

加密和解密用同一把密钥。

```
明文 + 密钥 → [加密] → 密文
密文 + 密钥 → [解密] → 明文
```

常见算法：AES、DES（已淘汰）、3DES、ChaCha20

**AES 举例：**
```java
SecretKeySpec key = new SecretKeySpec("1234567890123456".getBytes(), "AES");
Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
cipher.init(Cipher.ENCRYPT_MODE, key);
byte[] encrypted = cipher.doFinal("hello".getBytes());
```

- 优点：速度快，适合大数据量加密
- 缺点：密钥如何安全传输给对方是个问题（密钥分发难题）

---

## 2. 非对称加密

一对密钥：公钥加密，私钥解密（或私钥签名，公钥验签）。

```
明文 + 公钥 → [加密] → 密文
密文 + 私钥 → [解密] → 明文

数据 + 私钥 → [签名] → 签名值
签名值 + 公钥 → [验签] → 验证是否合法
```

常见算法：RSA、ECC（椭圆曲线）、DSA

- 优点：解决了密钥分发问题，公钥可以公开
- 缺点：计算开销大，比对称加密慢 100-1000 倍，不适合加密大数据

---

## 3. 核心对比

| 对比项 | 对称加密 | 非对称加密 |
|--------|----------|------------|
| 密钥数量 | 1个 | 2个（公钥+私钥） |
| 速度 | 快 | 慢 |
| 密钥分发 | 困难 | 简单（公钥可公开） |
| 典型算法 | AES、ChaCha20 | RSA、ECC |
| 适用场景 | 大数据加密 | 密钥交换、数字签名 |

---

## 4. 两者结合使用（混合加密）

这正是 HTTPS/TLS 的做法：

```
1. 非对称加密  →  安全交换对称密钥
2. 对称加密    →  用交换到的密钥加密实际数据
```

取长补短：非对称解决密钥分发，对称保证传输性能。

---

## 5. 数字签名 vs 加密的区别

很多人容易混淆：

- 加密：用公钥加密 → 只有私钥持有者能解密（保密性）
- 签名：用私钥签名 → 任何人用公钥验证（身份认证 + 防篡改）

HTTPS 证书里，CA 用自己的私钥对服务器证书签名，浏览器用 CA 公钥验签，确认证书没被伪造。
