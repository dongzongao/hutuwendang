# 时间复杂度与空间复杂度

## 1. 时间复杂度

衡量算法执行时间随数据规模增长的趋势，用大 O 表示法。

**常见复杂度从低到高：**

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
```

| 复杂度 | 名称 | 典型场景 |
|--------|------|----------|
| O(1) | 常数 | 数组下标访问、HashMap get |
| O(log n) | 对数 | 二分查找、平衡树操作 |
| O(n) | 线性 | 单层遍历、链表查找 |
| O(n log n) | 线性对数 | 快排、归并排序 |
| O(n²) | 平方 | 双层嵌套循环、冒泡排序 |
| O(2ⁿ) | 指数 | 递归求子集、暴力回溯 |

**代码示例对照：**

```java
// O(1) - 直接返回
int getFirst(int[] arr) { return arr[0]; }

// O(log n) - 每次缩小一半
int binarySearch(int[] arr, int target) {
    int l = 0, r = arr.length - 1;
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) l = mid + 1;
        else r = mid - 1;
    }
    return -1;
}

// O(n) - 单层循环
int sum(int[] arr) {
    int s = 0;
    for (int x : arr) s += x;
    return s;
}

// O(n²) - 双层嵌套
void bubbleSort(int[] arr) {
    for (int i = 0; i < arr.length; i++)
        for (int j = 0; j < arr.length - i - 1; j++)
            if (arr[j] > arr[j+1]) { int t = arr[j]; arr[j] = arr[j+1]; arr[j+1] = t; }
}
```

## 2. 空间复杂度

衡量算法运行时额外占用内存随数据规模增长的趋势。

| 复杂度 | 典型场景 |
|--------|----------|
| O(1) | 原地操作，只用几个变量 |
| O(log n) | 递归调用栈深度为 log n（如二分递归） |
| O(n) | 额外开一个数组、HashMap、递归深度为 n |
| O(n²) | 二维 DP 数组 |

```java
// O(1) 空间 - 双指针反转数组
void reverse(int[] arr) {
    int l = 0, r = arr.length - 1;
    while (l < r) { int t = arr[l]; arr[l++] = arr[r]; arr[r--] = t; }
}

// O(n) 空间 - 额外 HashMap
int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>(); // 额外 O(n)
    for (int i = 0; i < nums.length; i++) {
        int comp = target - nums[i];
        if (map.containsKey(comp)) return new int[]{map.get(comp), i};
        map.put(nums[i], i);
    }
    return new int[]{};
}

// O(n) 空间 - 递归调用栈
int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n-1) + fibonacci(n-2); // 栈深度 O(n)
}
```

## 3. 如何分析

**时间复杂度看循环层数：**
- 单层循环 → O(n)
- 双层嵌套 → O(n²)
- 每次减半 → O(log n)
- 单层 + 二分 → O(n log n)

**空间复杂度看额外开了什么：**
- 只有固定变量 → O(1)
- 开了和 n 相关的数组/Map → O(n)
- 递归深度 → 看递归树的深度

---

## 4. 常见误区

- 递归不一定是 O(n) 空间，二分递归是 O(log n)
- O(n log n) 的排序（归并、快排）比 O(n²) 冒泡快得多，n=10000 时差距是 13万 vs 1亿次操作
- 空间和时间可以互换，很多算法用空间换时间（如 DP、缓存）
